合约框架：anchor

创建anchor项目

  1. app:默认为空，用来放前端代码

  2. programs:合约代码。默认会有一个项目同名文件夹，里面有一个lib.rs文件

  3. tests:测试目录。默认包含一个当前项目名的测试

  4. migrations:合约的部署，迁移脚本

  5. Anchor.toml:项目的配置文件

    本地的合约地址([programs.localnet])

    合约仓库([registry])

    solana网络（[provider]）

    脚本([script])。脚本可以使用anchor run <script_name>运行，默认是test脚本

pda的概念和用法

pda：派生账户

pda的使用场景

方法1：hashmap

问题在于，当用户比较多时，账户有10M大小的限制，和web2不一样的是app是可以直接读账户，而不需要通过合约。而当app拉取account，Dapp当查询某一个人的计时器需要拉取全部数据

方法2：独立账户

给不同的wallet生成不同的counter账户

对于a和b比较麻烦，因为需要记住各自的地址。像网站的密码和用户名

方法3：PDA

优点：

  • 不需要记住钱包和账户的映射关系
  • 每个账户独立且不会冲突

还是不同的wallet生成不同的counter账户，但地址可能就是

wallet A pubkeys + program pubkey

wallet B pubkeys + program pubkey

账户的地址也必须是合法的32位的

在lib里加seed和bump指定账户为PDA账户

bump可以指定，但是不指定使用的就是匹配上的第一个
