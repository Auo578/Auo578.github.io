Solidity意义上的合约是代码（其 函数）和数据（其 状态）的集合， 驻留在以太坊区块链的一个特定地址。

uint storedData; 声明了一个名为 storedData 的状态变量， 类型为 uint （ unsigned integer，共 256 位）

（type）：unix

（）：该函数不接受任何参数

**public**：这是函数的可见性修饰符，表示该函数可以被任何外部账户或合约调用，也可以被合约内的其他函数调用。

**view**：这个修饰符表明该函数不会修改合约的状态（即不会改变任何状态变量的值），它只会读取存储在区块链上的数据。

诠释

Solidity合约也可以包含NatSpec注释。 它们用三重斜线（ ///）或双星号块（ /** … */）来写， 它们应该直接用在函数声明或语句之上。
建议 Solidity 合约使用 NatSpec 对所有公共接口（ABI 中的一切）进行完全注释。

标签 应用于
@title 一个应该描述合约/接口的标题 contract, library, interface
@author 作者的名字 contract, library, interface
@notice 向终端用户解释这个东西的作用 contract, library, interface, function, public state variable, event
@dev 向开发人员解释任何额外的细节 contract, library, interface, function, state variable, event
@param 就像在Doxygen中一样记录一个参数（必须在参数名之后） function, event
@return 记录一个合约的函数的返回变量 function, public state variable
@inheritdoc 从基本函数中复制所有缺失的标签（必须在合约名称之后） function, public state variable
@custom:... 自定义标签，语义由应用程序定义 everywhere

yarn init -y:
这个命令会初始化一个新的Node.js项目，创建一个package.json文件。-y选项意味着自动接受所有默认配置，而不需要用户交互。

yarn add -D hardhat:
这个命令将Hardhat作为开发依赖（-D等同于--dev）添加到项目中。Hardhat是一个用于智能合约开发、测试和部署的开发框架，通常用于以太坊相关的项目。

npx hardhat:
这个命令用于运行Hardhat的命令行工具。npx是一个包管理器工具，它会自动找到项目中安装的hardhat并执行。常用于运行Hardhat脚本或启动开发环境。如果你运行npx hardhat，通常会看到Hardhat的帮助信息，或者它会启动一个新的Hardhat项目。

Hardhat：合约审计机构，编译部署合约到任意以太坊网络的框架
